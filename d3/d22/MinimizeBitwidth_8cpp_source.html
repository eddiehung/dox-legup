6 //
7 // This pass shrinks arbitrary precision integer bitwidths from standard
8 // C sizes: i8, i16, i32, i64 to arbritrary precisions i31, i9 etc.
9 // Having a better idea of integer widths aids functional unit binding
10 //
11 // LLVM binary operations are limited to bit widths of 8, 16, 32, and 64.
12 // However, these widths don't reflect the real hardware because Quartus
13 // will optimize an operation's bit width. For example, given a 32 bit
14 // integer x, x *and* 60 only requires the lower 6 bits of x (the upper
15 // 26 bits are zero). If x is the result of a 32 bit addition with only
16 // one successor (the *and* operation) then the adder only requires 6
17 // bits. To calculate the required bit width of each operation we
18 // performed an iterative forward pass and a single backward pass. During
19 // the forward pass we look for add, and, right shift, and sign/zero
20 // extend operations. We set the bit width of each add to the max of its
21 // predecessor's bit widths plus one. For an *and* with a constant, the
22 // bit width is set to the smallest width that will accommodate the
23 // constant. For a right shift by a constant the bit width is equal to
24 // the input width minus the constant. Sign/zero extend operations have
25 // their bit width set to the width of their original operand before
26 // extension. The forward pass iterates until no more changes are made.
27 // Next there is a single backwards pass through the graph, where each
28 // operation's bit width is set to the maximum width of any of its
29 // successors.
30 //
