7 // Constraint Class for LP solver
8 // these functions help manage the LP constraints by keeping track
9 // of the row number for each constraint
10 // the big problem with the constraint row number is it changes every
11 // time you del_constraint on the LP
12 // the LP formulation for constraints are indexed by row number
13 // the index starts at 1
14 // we *only* keep track of the SDC resource conflict resolution
15 // constraints (i.e. loads/stores) because these might need to be modified
16 // during backtracking
17 //
18 //===----------------------------------------------------------------------===//
