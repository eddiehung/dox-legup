7 // A PatternMap is a container for Graph objects used by Binding.cpp.
8 // It organizes patterns of operations by isomorphic equivalence,
9 // taking commutative operations into account.
10 //
11 // Each Pattern (e.g. add->add) is represented by a vector of graphs. For example
12 // if the add->add pattern occurs with frequency 5 in a program, then the
13 // "add->add" pattern will be a vector with 5 graphs. Each of these
14 // is called a GraphList:
15 //
16 // GraphList = vector<Graph*>, representing all the
17 // graphs of that pattern
18 //
19 // Moreover, the graphs in a PatternList are all checked for isomorphic (not only
20 // topological) equivalence, by checking all commutative operations.
21 //
22 // Since there are many patterns which can exist (add->sub, mul->add, etc) a vector
23 // of Patterns, i.e. a vector of GraphLists. is required.
24 // This is represented by a PatternList:
25 //
26 // PatternList = vector<GraphList>
27 //
28 // For example a program may have 5 "add->add"s and 3 "add->sub"s, in which case
29 // the PatternList is of size 2, corresponding to a GraphList of size 5 and a
30 // GraphList of size 3
31 //
32 // Finally, for faster lookup, PatternLists are separated by size using a map:
33 // NOTE: lookup can be made even faster by also creating bins for the root
34 // operation (group all PatternLists by size and by root operation)
35 //
36 // PatternMap = map<int (pattern size), PatternList>
37 //
38 // E.g. PatternMap[1] contains all patterns of size 1, etc.
39 //
40 // Hence a PatternMap is:
41 //
42 // PatternMap = map<int, vector< vector<Graph*> > >
43 //
44 // However, the implementation is abstracted, so the only required operation to
45 // insert each Graph into a PatternMap is the Add() operation. This adds the
46 // graph to the required PatternList and GraphList, creating lists as necessary,
47 // and checking for isomorphic equivalence.
48 //
49 // Then, the iterators PatternMap_iterator, PatternList_iterator and
50 // GraphList_iterator can be used to iterate over every graph as needed.
51 //
52 //===----------------------------------------------------------------------===//
