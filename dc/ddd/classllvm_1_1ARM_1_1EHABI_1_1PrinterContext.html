ARM EHABI Section 6.2 - The generic model

An exception-handling table entry for the generic model is laid out as:

3 3 1 0 0 +-+---------------------------—+ |0| personality routine offset | +-+---------------------------—+ | personality routine data ... |

ARM EHABI Section 6.3 - The ARM-defined compact model

An exception-handling table entry for the compact model looks like:

3 3 2 2 2 2 1 0 8 7 4 3 0 +-+—+-—+--------------------—+ |1| 0 | Ix | data for pers routine | +-+—+-—+--------------------—+ | more personality routine data |

ARM EHABI Section 5 - Index Table Entries

  • The first word contains a PREL31 offset to the start of a function with bit 31 clear
  • The second word contains one of:
    • The PREL31 offset of the start of the table entry for the function, with bit 31 clear
    • The exception-handling table entry itself with bit 31 set
    • The special bit pattern EXIDX_CANTUNWIND, indicating that associated frames cannot be unwound
